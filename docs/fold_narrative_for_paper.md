# The Fold: 一个可学习、可验证的 Agent 工作记忆折叠机制
### （AAAI 投稿 · 方法叙事与创新点梳理）

---

## 0. 一句话故事

> 一个长时间工作的 agent，就像一个连续几天调试同一个 bug 的工程师：他不可能把每一条命令、每一屏输出、每一次试错都原封不动地记在脑子里。真正高效的工程师会**周期性地停下来，把"刚刚这一段"合上（fold），提炼成一小块结构化的工作记忆——目标是什么、试过什么、哪条路死了、下一步干什么——然后清空草稿纸，带着这块记忆继续**。我们把这个"折叠"动作变成一个**模型能学会、我们能验证、还能自我演化**的机制。

"Fold（折叠）"就是这篇论文的核心动词。

---

## 1. 为什么需要折叠：三重困境

Agentic LLM 在解决真实任务（修一个 GitHub issue、配置一台服务器、跑一条数据流水线）时，会进入一个 **观察→思考→行动→再观察** 的长循环。每一轮，模型的上下文都会追加：它发出的命令、工具返回的（往往很长的）输出、它的推理。于是上下文**单调增长**，带来三个同时恶化的问题：

1. **算力困境**：每一轮推理都要在越来越长的前缀上做一次前向。对全注意力模型是近似二次增长；对**线性注意力 / 混合架构**（本文的部署目标），由于跨请求的 prefix 缓存在这类递归状态上无法复用，**每一轮都要把整段历史重新 prefill**——上下文越长，每轮越贵。
2. **注意力稀释困境**：真正关键的少数事实（一个报错、一个行号、一个已确认的根因）被淹没在成千上万 token 的日志、目录列表、文件 dump 里。有效信息密度随轮次下降，模型开始"忘事"、重复撞同一堵墙。
3. **成本 / 上限困境**：上下文迟早撞到窗口上限或预算上限，长程任务被迫中断。

一个自然的想法是"**边走边压**"：在合适的位置，把一段原始历史换成一小段摘要。但——**朴素的摘要会悄悄杀死任务**。这正是本文要解决的核心张力，也是我们最关键的洞察所在。

---

## 2. 折叠的解剖：一次 fold 到底发生了什么

我们把 agent 的一条轨迹形式化为：任务锚点 `Q`（系统提示 + 任务描述）后面跟着一串决策步 `s_1, s_2, ...`，每一步是一个 `(thought, action, observation)` 三元组。

一次**折叠**发生在某个选定的步 `a_i` 上，做三件事：

1. **切分**：把当前工作区分成三段——
   - `Q`：任务锚点，**永远逐字保留**（它是一切的参照系）；
   - 一段**可压缩历史** `[lo, hi]`：这段将被移除；
   - 最近 `k` 步 `I^(k)`：保持原样（agent 需要即时的原始上下文来接着动手）。
2. **提炼**：把可压缩段压缩成一块**结构化工作记忆 `M`**（见 §4）。
3. **替换**：原始的 `[lo, hi]` 从上下文里**物理删除**，`M` 取而代之。工作区从 "`Q` + 一大段原始历史 + 最近步" 变成 "`Q` + `M` + 最近步"——**重新变得紧凑、有界**。

于是，推理时任意时刻的工作上下文都可以写成论文里那个干净的形式：

> **C(t) = ( Q, M(t), I^(k) )**

——一个**恒定量级**的三元组：任务、当前记忆、最近原始步。无论 agent 已经跑了 30 步还是 300 步，它面对的都是这个有界工作区。**这就是折叠的全部意义：把无界的历史压成有界的工作记忆，且训练时的样子和推理时完全一致。**

**关键设计——训练即推理**：我们不是在推理时外挂一个摘要器。我们把"折叠后的紧凑轨迹"直接做成训练数据，让模型**自己学会在该折的时候折、并亲手写出 M**。推理时它就是在复现训练时见过的模式：到了某个点，它自发地"合上历史、写一块记忆、继续"。折叠不是一个外部工具，而是模型**学到的一种行为**。

## 3. 第一个创新点：折叠触发的"去具身化"——从工具名到语义信号

**故事**：我们最初的折叠触发器是"看到测试通过就折""看到第一次写文件就折"。它在软件工程（SWE）数据上跑得很好——直到我们把模型放到更广的终端任务上，发现它**一次都不折叠**。诊断后真相很扎心：触发器绑死在了具体工具名和领域表象上——`pytest 通过`、`str_replace_editor 创建文件`、`Python traceback`。换一个 agent、换一个领域，这些表象根本不出现，折叠行为就"迁移不动"。**模型学到的不是"何时该折叠"，而是"看到 SWE 的某些字样就折叠"。**

**创新**：我们把折叠触发从**具身的工具表象**重写为**域无关的 agent 语义信号**：

| 语义信号 | 含义（任何 agent 域都成立） | 原来的具身版本 |
|---|---|---|
| `expansion` | 自上次折叠以来累计 token 超过预算 | （本就域无关，保留为主力） |
| `error_recovery` | 一串失败后回到一个干净状态 | 只认 Python traceback → 泛化到任意 CLI 报错 |
| `first_action` | **探索结束、首个改变状态的动作**（只读命令流 → 首个写/改） | 只认 `str_replace_editor`（SWE 编辑器）|
| `verified` | 一次检查/验证通过（先前失败的命令现在成功） | 只认 `pytest passed` |
| `plan` | 一个显式的规划里程碑 | （有则用，无则自然不触发）|

判定用的是**动作的语义**（只读 vs 改状态、报错 vs 干净、预算），而不是工具名——`error_recovery` 和 `expansion` 这两个最稳的信号完全域无关。我们还把折叠时模型吐出的那句触发语从 SWE 味（"Tests are passing…"）改成中性措辞（"I've made solid progress; I'll consolidate…"）。

**效果（可量化的创新点证据）**：仅仅把折叠点从"具身触发"换成"语义触发"，在同一套数据、同一预算下，**下游可用性（见 §5 的充分率）从 57.6% 跳到 73.6%（+16 个百分点）**——而且这**同时**打通了跨域迁移。一石二鸟：既让折叠行为可迁移，又让折叠质量更高。深层原因很美：语义信号天然落在**干净的任务边界**上（探索刚结束、验证刚通过、错误刚恢复），在这些点压缩是**接近无损**的；而"攒够预算就地切"的 `expansion` 折在半途，是最难压的（我们的分信号分析证实：`first_action` 82% 充分、`expansion` 只有 69%）。**折叠点选得好，胜过把摘要写得好。**

## 4. 第二个创新点：什么样的记忆值得保留——结构化 + 前向可执行

一块好的 `M` 不该是一段流水账摘要。我们把它设计成**七段式结构化记忆**，分成"回顾"与"前向"两半：

**回顾四段（记录过去）**
- `[Goal]`：这一段在追的子目标
- `[Attempts & Results]`：试过的策略、结果、已确认的根因 / 当前假设 / 被证伪的假设
- `[Environment Feedback]`：关键环境反馈——报错、路径、配置、结果
- `[Active Constraints]`：仍然约束后续的不变量 / 契约

**前向三段（指向未来）——这是我们的关键创新**
- `[Next]`：下一步的**计划级**子目标（"用先前失败的那个测试验证修复"），**绝不写具体命令**
- `[Open Threads]`：试了但没做完的子目标、待验证的假设
- `[Dead Ends]`：已被证伪/报错的做法，写成祈使式护栏（"不要再试 X——因为 Y"）

**为什么前向段是创新而非锦上添花**：传统的"折叠"只是压缩过去。但 agent 折叠后最痛的，是**重新推导计划**——它把探索合上了，却也把"我接下来要干什么"忘了。`[Next]`/`[Open Threads]` 让折叠后的模型**不必重推计划**（这正是"复用压缩表示"应有之义）；`[Dead Ends]` 直接**掐断重复撞墙的死循环**（我们在终端任务里亲眼见过 agent 反复跑同一条失败命令）。

**一个必须解决的技术难点——前向段怎么生成而不"泄题"**：摘要器是离线的，它能看到折叠点**之后**真实发生了什么，所以能写出 `[Next]`。但如果它把未来的具体命令逐字抄进 `[Next]`，模型就会退化成"背诵 gold 路径"，学不到规划能力。我们的约束是**计划层 vs 动作层**的严格分离：`[Next]` 只能写"意图"（做什么、怎么验证），**具体动作永远留给模型自己选**。这样既保留了学习信号，又注入了前向引导。

**锚点结转（Pinned facts）**：某些事实（工作区路径、核心文件位置、任务不变量）在整个任务里恒定不变。我们把它们标记为 `[Pinned]`，**每一次折叠都逐字结转、绝不进入压缩**。这是对抗"迭代摘要信息衰减"的第一道防线。

## 5. 第三个创新点（本文的理论核心）：折叠的"可验证充分性"

这是整篇论文最该突出的地方。

**故事**：我们一度以为，只要教模型"写 M"就够了。但在 teacher-forcing 训练下有个致命的漏洞——**折叠点之后的"正确续写"永远是给定的**，所以哪怕 M 写成一段漂亮的废话，训练损失也照样很低。模型学到的可能是"写好看的摘要"，而不是"写有用的摘要"。**我们怎么知道一块 M 到底够不够用？**

**洞察**：一块**充分**的记忆，其判据不该是人工规则，而应是——**M 是否是"过去"对于"未来"的一个充分统计量**。也就是说，给定 M，未来该做什么，应该和给定完整原始历史时**一样可预测**。

**形式化（反事实 NLL 检验 = 条件互信息的蒙特卡洛估计）**：对一个压缩 `[lo, hi]`、发生在步 `s` 的折叠，取折叠点之后的 `T` 个 gold 决策步 `C` 作为"未来"。构造两个共享同一前缀的上下文：

```
RAW    = ( 前缀, 原始 steps[lo..hi], C )        # 完整历史
FOLDED = ( 前缀, context-call(M), ack, C )       # M 取代 [lo..hi]
```

用**将要微调的那个基座模型自己**（不是更强的老师——理由见下），只在 `C` 的 assistant/action token 上计算负对数似然，得到：

> **ΔNLL_per_tok = ( NLL(C | FOLDED) − NLL(C | RAW) ) / |C 的 assistant token|**

- **ΔNLL ≈ 0** → 折叠没有损失对未来的预测力 → **M 是充分统计量** → 保留；
- **ΔNLL ≫ 0** → 折叠丢掉了续写所需的信息 → **不充分** → 丢弃或定向修复。

这本质上是在估计折叠所摧毁的条件互信息 `I(C; RAW) − I(C; M)`。**它用模型自身的分布来定义"够不够用"，比任何人工启发式都更本质**——把"压缩质量"从一个主观、不可度量的东西，变成一个**可计算、可作为验收闸门的标量**。

**为什么用基座而非强老师打分**：充分性必须对**将来真正跑推理的那个模型**校准。一个事实对 27B 老师显然、对 9B 学生不显然，那 M 就**必须替 9B 写出来**。所以打分器是学生本身——这保证了我们筛出的记忆是"对最终执行者充分"，而非"对一个更聪明的旁观者充分"。

**闭环修复**：当 ΔNLL 高时，我们知道具体是 `C` 的哪些 token 概率暴涨——即缺了什么。第二遍摘要器被喂进真实续写 `C` 并被告知"你漏了产出这些步骤所需的信息，补上路径/行号/报错/结论，但不要逐字照抄"。防作弊闸门：M 不得逐字包含 `C` 的动作串（否则会平凡地降低 NLL 却毁掉压缩）。

**创新点的分量**：据我们所知，这是**第一次把"agent 记忆压缩的质量"表述为一个信息瓶颈意义下的充分统计量判据，并用它作为数据构造的验收闸门**。它把折叠从"启发式摘要"提升为"有可验证保证的压缩"。而且它**可迁移地**解释了为什么折叠点很重要（§3）：好的折叠点让 M 更容易成为充分统计量。

## 6. 第四个创新点：记忆的递归演化——单块、可维护、不重建

**故事**：论文里的记忆是 `C(t) = (Q, M(t), I^k)` —— 一块**单一的、随时间演化**的记忆 `M(t)`。但朴素的做法是每次折叠都独立地压一段窗口，得到一堆互不相关、并存累积的 M。那不是"维护一块记忆"，那是"不断新建便签"。

**创新**：我们把折叠改成**递归滚动**——

> **M_i = compress( M_{i-1} + 自上次折叠以来的新原始步 )**

每一次折叠的输入是"**上一块记忆 + 新步**"，输出是**更新后的单块记忆**：结转仍相关的、整合新发现的、丢掉已解决/过时的。工作区始终是 `(Q, M_i, 最近步)`，量级恒定。模型学到的不再只是"写摘要"，而是**"合并记忆"这个动作本身**——carry-forward + integrate + drop-resolved。这才是"复用压缩表示"的完整含义，也才真正匹配论文的单块演化 `M(t)`。

**递归的头号风险——信息衰减**：迭代摘要的通病是"每次丢一点，五轮后早期事实全没了"。我们用三道防护把它上界住：
1. **§5 的充分性检验放在每个折叠上** → 每一步的信息损失被显式上界卡住（这就是充分性与递归**必须一起用**的原因——递归没有充分性检验会漂移）；
2. **`Q` 永远逐字保留**，当锚；
3. **Pinned facts 逐字结转**，关键事实不进压缩。

**训练格式（training == inference 的极致体现）**：每一条训练样本就是**一次滚动**——

```
[ Q, ctx(M_{i-1}), ack, 被折叠的新原始步, ctx(M_i)←监督目标, 后续续写←也训练 ]
```

损失只在 `M_i` 的生成 + 其后的续写上（前面的 M_{i-1} 和被折原始步是上下文、被 mask）。于是模型同时学两件事：**(a) 看着旧记忆+新步，产出合并后的新记忆；(b) 拿着新记忆正确地继续**。每条样本都是一个**有界**窗口，与轨迹总长无关——我们实测：即使最长的源轨迹（~90k token 原始历史），其每一个滚动样本也 ≤ 45k token，稳稳落在训练窗口内。**折叠让"任意长的 agent 轨迹"变成"一串有界的、可训练的记忆-维护样本"。**

## 7. 第五个协同点：折叠与线性注意力架构的天然契合

这是一个漂亮的、可写进 discussion 的观察。对我们部署的**混合线性注意力**模型（大部分层是 gated-delta 线性注意力 + 少量全注意力），跨请求的 prefix KV 缓存在递归状态上**结构性地无法复用**（每一轮都要重新 prefill 整段上下文）。

在这种架构上：

| | 全注意力模型 | 线性注意力 / 混合模型 |
|---|---|---|
| 跨轮 prefix 缓存 | 有，折叠会移除中间→后缀缓存全失效（**折叠有惩罚**）| 本就没有可复用的前缀缓存 |
| 折叠的净效应 | 省内存但抵消部分缓存收益 | **纯赚**：每轮重新 prefill 的序列直接变短 |

也就是说，**在线性注意力模型上，折叠没有"破坏缓存"的惩罚，反而每一轮都省下重算的 token**——折叠与这类高效长上下文架构是**协同**的。这为"为什么在这类模型上做折叠特别值得"提供了一个架构层面的论据。

## 8. 把五点串成一条完整的方法流水线

```
① 折叠点选择（语义信号，域无关）
     │  产出：fold points [lo,hi] + signal
     ▼
② M 生成（结构化 + 前向 + pinned）
     │  ├─ 七段 schema：回顾四段 + 前向三段([Next]/[Open]/[Dead Ends])
     │  ├─ 前向段喂折叠后续写、强制计划级（防泄题）
     │  └─ 递归输入 = (上一块 M + 新步)，pinned 逐字结转
     ▼
③ 充分性验收（反事实 ΔNLL，学生基座打分）
     │  过 → 保留；不充分 → 定向修复（闭环）或丢弃
     ▼
④ 拼接成"训练即推理"的滚动样本
     │  单块滚动 M；masking：只在 M_i 生成+续写上算 loss
     ▼
   有界工作区数据集 → 微调 → agent 学会"在该折时折、亲手写 M、带记忆续写"
```

**协同关系一句话**：**语义触发**让折叠**选对位置**（§3）→ **前向 schema** 让 M **装上计划**（§4，供货）→ **递归**让 M **学会被维护**（§6，演化）→ **充分性检验**在每一步**验证这块 M 真的扛得住续写**（§5，质检 + 防递归漂移）。缺了充分性检验，前向和递归都可能写出"好看但没用"的 M；有了它，一切收益才落地。

## 9. 面向 AAAI 的创新点定位（建议怎么写）

把"Fold"作为核心贡献，建议这样组织 novelty：

1. **主贡献 / 理论核心**：*可验证充分性折叠*——首次将 agent 工作记忆压缩表述为**信息瓶颈下的充分统计量**判据（反事实 ΔNLL / 条件互信息），并将其作为**数据构造的验收闸门**，把"折叠"从启发式摘要提升为有保证的压缩。（§5）
2. **贡献二**：*域无关的语义折叠触发*——把折叠行为从具身工具表象解耦，实现跨 agent 域迁移；实证 +16 个百分点的下游可用性提升。（§3）
3. **贡献三**：*前向、递归、可维护的单块记忆*——`[Next]/[Dead Ends]` 前向段 + 递归滚动 + pinned 结转，配合充分性检验对抗迭代衰减；"训练即推理"的有界样本构造。（§4, §6）
4. **贡献四 / discussion**：*折叠与线性注意力架构的协同*——在无法复用 prefix 缓存的高效长上下文模型上，折叠无缓存惩罚、每轮净省 prefill。（§7）

**一句话 abstract 种子**：
> We recast agentic context compression as *folding*: at semantically-triggered, domain-general checkpoints, an agent collapses its raw history into a single, recursively-maintained, forward-looking memory. Crucially, we do not merely summarize—we **verify** each fold as a *sufficient statistic* of the past for the future via a counterfactual next-token-likelihood test, and keep only folds that provably preserve the agent's ability to continue. This turns memory compression from a heuristic into a measurable, gated guarantee, yielding bounded working context, cross-domain transfer, and—on linear-attention backbones—free per-turn prefill savings.

## 10. 我们已经拿到的、可作为论据的实证（真实数据）

- **域无关触发的增益**：同数据同预算下，折叠点从具身 → 语义，下游充分率 **57.6% → 73.6%（+16pt）**，median ΔNLL 0.123 → 0.087。
- **分信号充分率**：`first_action` 82%（探索→动作的干净边界）> `verified` 73% ≈ `error_recovery` 72% > `expansion` 69%（任意切，最差）——**证明折叠位置决定质量**。
- **充分性的必要性**：仅"加大 M 预算"从 57.6% 只到 62.1%（老师自限于 ~0.2 压缩比，逼不大）；"闭环修复"也只到 ~62%——**证明问题不在 M 大小，而在选对内容/选对位置**，充分性闸门是正解。
- **失充分的根因（内容分析）**：不充分折叠集中在**探索/阅读密集**的切片（file-view 多、原始 obs 大 25%），而**结果型**内容（报错/编辑/测试/结论）压缩近乎无损——**这是一个域无关的 agent 现象，不是 SWE 特有**。
- **有界性实证**：最长源轨迹（~90k token）的每个折叠样本 ≤ 45k token，全部落在训练窗口内——**折叠让任意长轨迹变成有界样本，一条不砍**。
